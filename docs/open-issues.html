#1 — Physical DI channel assignment for SepSol2/3 + PushSol2/3 Blocking hw

Owner: electrical integrator · Blocks: full-rig sensor feedback · DO side fully assigned; software already wired

Question. Which GVL_IO.dIn[] array slots are wired to the physical proximity switches of the four unassigned pistons?

Current state. GVL_IO.dIn and GVL_IO.dOut are sized ARRAY[1..16]. The SOL-Spring migration packed all six Extend coils into dOut[1..6], so DO is fully assigned. DI still needs SepSol2/3 + PushSol2/3.

dOut[7] and dOut[8] — previously PushSol1's SOL-SOL retract coil pair — are freed by the SOL-Spring migration and available for reuse. Slots 9..16 on both DI and DO are unassigned.

What "soft-only" means today. All six actuators drive their coil correctly (the DO layout is fully populated). What's missing on SepSol2/3 + PushSol2/3 is the Extended / Retracted position feedback: the FB's bIsExtended / bIsRetracted stay FALSE until the operator flips the Simulate checkbox on the corresponding piston tile — or on the AutoCycle page via bSimStep_*.

How to resolve. Once channels are assigned:

  1. Fill in the four TBD rows above.
  2. Open Flower_PLC_HMI/PLC1/POUs/MAIN.TcPOU. Eight lines in the READ INPUTS block are commented placeholders like // GVL_App.bSensorSepRetracted2 := GVL_IO.dIn[??];.
  3. Replace each [??] with the actual slot number, uncomment the line, and remove its TODO marker.
  4. Rebuild + Activate. In the Watch window, force GVL_IO.dIn[N] for each new switch and confirm the corresponding bIsExtended / bIsRetracted indicator on the HMI updates.

Tag convention. <Device>Sol<N> is a single Extend coil (SOL-Spring); the retract sensor still exists on the DI side because the retracted-position feedback is useful even without a retract coil.

Related caveat. After the array grew from [1..8] to [1..16], the EtherCAT links for slots 1..8 must be re-verified in the System Manager — TwinCAT does not always re-attach linked variables when their parent array is resized. Symptom: SepSol1 / PushSol1 sensor indicators stop updating even though nothing else changed.

✓ Full-machine auto sequence Closed

Landed as FB_MasterAutoCycle — see CLAUDE.md DECISION section

Ten-state coordinator across 1..3 pairs of (SepSolN, PushSolN) pistons plus the robot's Pos1/Pos2 signals. Outbound AUTO_STARTED / PUSH_DONE / PISTONS_ERROR pulses; inbound RESET_ERROR. INT error code (0=OK, 1..5 = step timeout, 99 = STOP-while-running). Own HMI page (AutoCycle.content) with 9 per-step Sim checkboxes for sensorless bench testing.

✓ SOL-Spring piston migration + coil latch fix Closed

All six pistons retyped to FB_SolSpringPiston_2Pos; DO layout packed into dOut[1..6]

SOL-Spring topology: single Extend coil, mechanical spring returns the piston. The Section 2 ladder is intentionally latch-and-retain (no ELSE branch) so the coil stays energised until the next command — a plain "clear when target reached" branch would let the spring pull the piston back mid-cycle. The retracted-position DI sensor still exists (position feedback), but no retract coil is driven.
